# 機能案：テスト前総復習モード

> ⚠️ **エージェントへ：** この機能は構想段階です。ユーザーが明示的に実装を指示するまで実装しないでください。

**目的：** 定期テスト・模試の直前に、復習予定日に関係なく「試験範囲の間違いノートを全部」まとめて復習できるモードを提供する。

---

## 概要（UX）

- 復習の入口に「**今日の復習**」（既存: `next_review_at <= today`）と「**範囲を選んで総復習**」の2モードを置く
- 総復習モードでは 科目 →（任意で）単元を複数選択 → ステータス（苦手中のみ / 克服済みも含む）を選んで開始
- 復習フロー自体（問題表示 → 自己採点 → 正誤記録）は既存の復習画面をそのまま再利用する

「テスト前に間違いノートを見返す」という紙の間違いノートの一番の使いどころをデジタルで再現する機能。

---

## API/DB 変更点

既存 API でほぼ実現できる。不足しているのは **mistake-notes 一覧の範囲フィルタ**のみ。

| 種別 | 変更 |
|------|------|
| API | `GET /v1/mistake-notes` に `subject_id` / `unit_id`（複数指定可）/ `status` クエリを追加（question 経由で JOIN して絞り込み） |
| DB | 変更なし |
| フロント | 復習入口のモード選択 UI＋範囲選択フォーム（科目・単元セレクトは登録画面の実装を再利用） |

---

## 実装ステップ

1. `GET /v1/mistake-notes`（`app/routers/mistake_notes.py`）に `subject_id` / `unit_id` / `status` フィルタを追加
2. `openapi.json` 再生成 → `npm run generate`
3. 復習入口画面にモード選択と範囲選択フォームを追加
4. 選択結果を既存の復習フローに渡す（出題リストが違うだけで画面は共通）

---

## 検討事項・トレードオフ

- 総復習中の解答記録も通常どおり `correct_streak` を動かす（既存仕様のまま）。短期間に連続正解すると `mastery_suggested` が出やすくなるが、「テスト前に3連続正解できたなら克服提案が出る」のはむしろ自然なので特別扱いしない
- 出題順: まずは登録順で十分。`wrong_count` の多い順（苦手優先）は検討の価値あり
- 範囲選択の複数単元指定は `unit_id` をカンマ区切り or 複数クエリパラメータで受ける（FastAPI の `list[UUID] = Query(None)`）
