> ⚠️ **エージェントへ：** このフォルダにある機能はすべて**構想段階**です。ユーザーが明示的に実装を指示するまで、**いかなる機能も実装しないでください**。ドキュメントの更新・追記の指示があった場合のみ、ドキュメントを編集してください。

**目的：** misnote の「このアプリならではの強み」を作るための機能案を集約する。各案は「概要+実装方針」レベルで記述し、採用が決まった機能のみ `docs/design/` の既存スペックと同等の詳細度に落とし込む。

---

## このアプリの個性（差別化の軸）

misnote は単なる暗記カードアプリ（Anki 等）ではなく、**「なぜ間違えたか」を記録して克服する**アプリ。差別化は次の3軸で考える。

1. **入口の軽さ** — 間違えた瞬間に負担なく記録できる（クイック保存・写真登録・プリセットマスタ）
2. **「なぜ」の構造化** — 間違いの原因をデータにして弱点を可視化する（原因タグ・統計）
3. **克服までの伴走** — 復習のタイミングと範囲をアプリが支援する（復習日提案・総復習モード・リマインダー）

## おすすめの実装順

1. **quick-save** — ユーザー起案。コスト小で「入口の軽さ」が即改善する
2. **preset-subjects-units** — ユーザー起案。初期設定の負担をなくす
3. **review-interval-suggestion** — DB変更なしで「伴走」体験が生まれる
4. **mistake-reason-tags** — 最大の差別化。統計の土台にもなる
5. **cram-mode** — 既存APIの小さな拡張で実現できる
6. **stats-dashboard** — タグ導入後にやると価値が跳ね上がる
7. **photo-upload** — 実用性は大きいがストレージ設計が絡む
8. **ai-assist** — API費用と品質検証が必要
9. **reminder-notification** — 認証（Phase 3）完了後に着手
