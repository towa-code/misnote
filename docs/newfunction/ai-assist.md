# 機能案：AI補助（理由の言語化・類題生成）

> ⚠️ **エージェントへ：** この機能は構想段階です。ユーザーが明示的に実装を指示するまで実装しないでください。

**目的：** LLM を使って (1) 「なぜ間違えたか」の言語化を手伝う、(2) 克服確認用の類題を生成する。あくまで**補助**であり、最終的な言語化・判断はユーザー自身が行う設計にする（自分で書くことに学習効果があるため）。

---

## 概要（UX）

### 1. 間違えた理由の言語化補助

- 登録/編集画面で「AIに聞いてみる」ボタン → 問題文・自分の答え・正解を渡し、「考えられる間違いの原因」を2〜3個の候補として提示
- ユーザーは候補を選んで**編集してから** memo に反映する（そのまま保存させない。原因タグの候補提示にも使える）

### 2. 類題生成

- 間違いノート詳細で「似た問題を出して」→ 同じ論点の類題を1問生成し、その場で解いてみる
- 生成問題は保存せず使い捨て（保存すると品質保証やデータモデルの問題が増えるため、まずは表示のみ）

---

## 設計方針

- Claude API を利用。用途が短文生成なので軽量モデル（Haiku クラス）でコストを抑え、品質不足なら上位モデルに切り替え
- API キーは既存の `app/config.py`（pydantic-settings）に `ANTHROPIC_API_KEY` を追加して `.env` 管理
- 同期リクエストで十分（数秒）。フロントはローディング表示

---

## API/DB 変更点

| 種別 | 変更 |
|------|------|
| API | `POST /v1/ai/suggest-reasons`（question_id または問題文/解答を受け、原因候補の配列を返す）/ `POST /v1/ai/similar-question`（question_id を受け、類題テキストを返す） |
| DB | 変更なし（生成結果は保存しない） |
| 依存 | `anthropic` SDK — **新規依存＋API利用料が発生するため導入前にユーザー承認** |
| フロント | 登録/編集画面と詳細画面にボタン＋候補表示 UI |

---

## 実装ステップ

1. `app/services/ai.py`（プロンプト組み立て＋API呼び出し）と `app/routers/ai.py` を追加、`main.py` にマウント
2. `config.py` に `ANTHROPIC_API_KEY`（未設定なら AI エンドポイントは 503 を返す＝機能ごと無効化できるように）
3. `openapi.json` 再生成 → `npm run generate`
4. フロントの UI（候補の選択→編集→memo 反映のフロー）

---

## 検討事項・トレードオフ

- **コスト管理:** ユーザー単位のレート制限（例: 1日20回）を最初から入れる。無制限だと事故る
- **正確性:** 特に類題生成は誤った問題・解答を出しうる。「AIが作った問題です。答えが正しいか確認してください」の注意書きを必ず表示。学生向けであることを踏まえプロンプトで難易度・語彙を制御
- **学習効果との両立:** 理由の言語化を丸投げさせない UI（候補は短いヒントに留め、編集必須にする）が本機能の肝
- 写真登録（[photo-upload](./photo-upload.md)）と組み合わせた OCR（写真→問題文テキスト起こし）は有力な拡張。Claude API の画像入力でそのまま実現できる
