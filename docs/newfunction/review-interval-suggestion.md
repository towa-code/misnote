# 機能案：復習日の自動提案

> ⚠️ **エージェントへ：** この機能は構想段階です。ユーザーが明示的に実装を指示するまで実装しないでください。

**目的：** 忘却曲線に沿った次回復習日をアプリが提案し、ワンタップで設定できるようにする。「いつ復習すればいいか分からない」問題を解消しつつ、復習日を自分で決めるという現行思想（手動設定）は崩さない。

---

## 概要（UX）

- 復習画面で正解/不正解を記録した直後に「**次回は◯月◯日（3日後）がおすすめ**」というチップを表示し、タップすると `next_review_at` に設定される。無視して手動の日付ピッカーで選ぶことも、設定しないこともできる
- 登録画面の復習日欄にも「+1日 / +3日 / +7日」のクイックチップを置く（これはフロントだけで完結する）

既存の `mastery_suggested`（3連続正解で克服を「提案」するだけで自動遷移しない）と同じ設計思想：**アプリは提案し、決めるのはユーザー**。

---

## 提案ロジック

`correct_streak` に応じて間隔を伸ばす（忘却曲線ライクな単純ステップ。SM-2 のような本格 SRS はやらない）:

| correct_streak（解答後） | 提案間隔 |
|---|---|
| 0（間違えた直後） | 翌日 |
| 1 | 3日後 |
| 2 | 7日後 |
| 3以上 | 14日後（このとき既存の mastery_suggested も出る） |

定数 `SUGGEST_INTERVALS = [1, 3, 7, 14]` として `routers/attempts.py` の `MASTERY_THRESHOLD` と並べて定義する。

---

## API/DB 変更点

| 種別 | 変更 |
|------|------|
| API | `AttemptResponse` に `suggested_next_review_at: date \| None` を追加（note が無い場合は null）。`create_attempt` 内で解答後の `correct_streak` から計算 |
| 設定 | 提案の適用は**既存の** `PUT /v1/mistake-notes/{id}`（next_review_at 更新）を使う。新エンドポイント不要 |
| DB | 変更なし |
| フロント | 復習画面（未実装の `/review`）に提案チップ、登録画面にクイックチップ |

---

## 実装ステップ

1. `AttemptResponse` スキーマに `suggested_next_review_at` を追加し、`create_attempt` で算出
2. `openapi.json` 再生成 → `npm run generate`
3. 復習画面の解答結果 UI に提案チップ＋ `PUT /v1/mistake-notes/{id}` 呼び出し
4. 登録画面に「+1日/+3日/+7日」チップ（フロントのみ）

---

## 検討事項・トレードオフ

- 間隔の数値（1/3/7/14）は根拠が経験則なので、定数として一箇所にまとめ後から調整可能にしておく。ユーザーごとの間隔カスタマイズは作らない（YAGNI）
- 本格 SRS（SM-2 等の ease factor 学習）に進むなら `mistake_notes` にカラム追加が必要になるが、「自分で復習日を決める」というアプリの個性と競合するため、提案止まりがこのアプリらしい
- 間違えた直後（翌日提案）は `wrong_count` に依らず一律とする。悪化するほど短くする案もあるが、まずは単純に
